# AGENTS.md — FOC_V0_2 (STM32G474CBT6)

## Project Overview
- **MCU**: STM32G474CBT6 (Cortex-M4 + FPU, 128K Flash / 128K RAM)
- **Purpose**: FOC (Field Oriented Control) motor controller firmware — early stage, skeleton only
- **Generated by**: STM32CubeMX (`FOC_V0_2.ioc`)
- **Build system**: CMake + Ninja, cross-compiled with `arm-none-eabi-gcc`

## Build Commands
```
cmake --preset Debug        # configure (outputs to build/Debug/)
cmake --build --preset Debug  # build → build/Debug/FOC_V0_2.elf
```
- Default build type is `Debug` (`-O0 -g3`). Release uses `-Os -g0`.
- VS Code CMake extension uses custom `cube-cmake` path — plain `cmake` CLI works if `arm-none-eabi-gcc` is on PATH.
- `compile_commands.json` is generated at `build/Debug/compile_commands.json` for clangd/intellisense.

## Agent Rules
- **不要主动运行 cmake 编译或构建命令** — 用户通过 VS Code STM32CubeIDE 插件构建，agent 只负责写代码

## Directory Structure
| Path | Purpose |
|---|---|
| `Core/Inc/`, `Core/Src/` | CubeMX 生成的代码（不要手动修改非 USER CODE 区域） |
| `Core/App/Inc/`, `Core/App/Src/` | 应用层：业务逻辑、算法、协议 |
| `Core/Drv/Inc/`, `Core/Drv/Src/` | 驱动层：传感器、执行器、外设抽象 |
| `Core/Bsp/Inc/`, `Core/Bsp/Src/` | 板级支持：GPIO、UART、SPI 等硬件初始化封装 |
| `Drivers/STM32G4xx_HAL_Driver/` | STM32 HAL 驱动源码（不要编辑） |
| `Drivers/CMSIS/` | ARM CMSIS 头文件（不要编辑） |
| `cmake/` | 工具链文件 |
| `cmake/stm32cubemx/CMakeLists.txt` | CubeMX 自动生成（不要修改） |
| `build/` | 构建输出（gitignore） |

## Critical CubeMX Conventions
- **NEVER edit outside `/* USER CODE BEGIN */` / `/* USER CODE END */` blocks** in `Core/` files — CubeMX will overwrite them on regeneration.
- Add new user sources to `CMakeLists.txt` (root) in the `target_sources()` / `target_include_directories()` / `target_link_libraries()` sections. Do NOT modify `cmake/stm32cubemx/CMakeLists.txt` — it is regenerated.
- Peripheral init order in `main.c`: `HAL_Init()` → `SystemClock_Config()` → `MX_*_Init()` (GPIO, SPI1, USART1).

## Configured Peripherals
- **SPI1** — SPI bus (pins not yet assigned in skeleton)
- **USART1** — UART (for debug/comm)
- **GPIO** — pin configuration

## Clock Configuration
- HSI oscillator → PLL → 170 MHz SYSCLK (`PLL_N=85`, `PLL_M=4`, `PLL_P=2`)
- Voltage scaling: `PWR_REGULATOR_VOLTAGE_SCALE1_BOOST`
- Flash latency: 4 wait states

## Toolchain Details
- CPU flags: `-mcpu=cortex-m4 -mfpu=fpv4-sp-d16 -mfloat-abi=hard`
- Linker script: `STM32G474XX_FLASH.ld` (128K Flash @ 0x08000000, 128K RAM @ 0x20000000)
- Defines: `USE_HAL_DRIVER`, `STM32G474xx`
- C standard: C11 (required — generated code won't compile with C99)
- Linker removes unused sections (`--gc-sections`); map file output enabled

## clangd / IDE Setup
- `.clangd` points to `build/Debug` compilation database
- VS Code uses `stm32cube-ide-clangd` extension with `cube` bundle
- `c_cpp_properties.json` references `compile_commands.json`
