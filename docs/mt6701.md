# MT6701 磁编码器驱动文档

## 1. 通信协议和时序

MT6701 使用 **SSI（Synchronous Serial Interface）** 协议通信，属于三线 SPI（只需 CSN、SCK、DO）。

### 引脚定义

| 引脚 | MCU 引脚 | 功能 |
|------|----------|------|
| CSN  | PB0      | 片选（低电平有效） |
| SCK  | PA5      | 时钟信号（MCU 输出） |
| DO   | PA6      | 数据输出（MT6701 → MCU） |
| MOSI | PA7      | 未使用（硬件短接至 MISO，需配置为输入） |

### 时序说明

- **起始条件**：CSN 下降沿启动一次读取
- **时钟极性**：CPOL = 0（空闲时 SCK 为低电平）
- **数据变化**：DO 信号在 CLK **上升沿**发生变化
- **数据采样**：建议在 CLK **下降沿**采集数据（SPI Mode 1）
- **数据长度**：每次读取 **24 位**（3 字节），共 24 个时钟周期
- **结束条件**：CSN 拉高结束本次通信

### 时序流程

```
CSN  ──┐         ┌──────────────────────────────────────────┐
       └─────────┘                                          │
                                                           │
SCK  ──────┐  ┌──┐  ┌──┐  ┌──┐       ┌──┐  ┌──┐  ┌──┐    │
           └──┘  └──┘  └──┘  └── ... ┘  └──┘  └──┘  └──   │
           ↑   ↑   ↑   ↑              ↑   ↑   ↑   ↑       │
           1   2   3   4     ...     21  22  23  24        │
                                                           │
DO   ──[D13][D12][D11]...[D0][M3][M2][M1][M0][C5]...[C0]──│
     ←─── 14位角度 ──→←─ 4位状态 ─→←── 6位CRC ──→         │
```

## 2. SPI 配置

使用 STM32G474 硬件 SPI1，配置参数如下：

| 参数 | 配置值 | 说明 |
|------|--------|------|
| Mode | Master | 主机模式 |
| Direction | RXONLY | 仅接收（MOSI 不输出，避免短接干扰） |
| DataSize | 8 bit | 每次传输 8 位 |
| CPOL | Low (0) | 空闲时时钟为低电平 |
| CPHA | 2EDGE (1) | 下降沿采样（Mode 1） |
| NSS | Soft | 软件管理片选（PB0 GPIO） |
| BaudRatePrescaler | 64 | 170MHz / 64 = **2.656 MHz** |
| FirstBit | MSB | 高位优先 |
| TIMode | Disable | 非 TI 模式 |
| CRCCalculation | Disable | 不使用硬件 CRC |

### 关键配置代码

```c
/* CubeMX 默认配置 */
hspi1.Init.Mode = SPI_MODE_MASTER;
hspi1.Init.Direction = SPI_DIRECTION_2LINES;
hspi1.Init.DataSize = SPI_DATASIZE_8BIT;
hspi1.Init.CLKPolarity = SPI_POLARITY_LOW;
hspi1.Init.CLKPhase = SPI_PHASE_2EDGE;
hspi1.Init.NSS = SPI_NSS_SOFT;
hspi1.Init.BaudRatePrescaler = SPI_BAUDRATEPRESCALER_64;
hspi1.Init.FirstBit = SPI_FIRSTBIT_MSB;

/* 用户覆盖（SPI1_Init 2 块） */
hspi1.Init.Direction = SPI_DIRECTION_2LINES_RXONLY;
hspi1.Init.CLKPhase = SPI_PHASE_2EDGE;
HAL_SPI_Init(&hspi1);

/* PA7 (MOSI) 重新配置为输入，消除短接总线竞争 */
GPIO_InitTypeDef gpio = {0};
gpio.Pin = GPIO_PIN_7;
gpio.Mode = GPIO_MODE_INPUT;
gpio.Pull = GPIO_PULLUP;
HAL_GPIO_Init(GPIOA, &gpio);
```

### 初始化顺序

```
MX_GPIO_Init()        → PB0 (CS) 输出
MX_USART1_UART_Init() → 串口调试
MX_SPI1_Init()        → SPI1 外设 + PA5/PA6/PA7 复用功能 + 覆盖配置
mt6701_init()         → DWT 初始化 + CSN 拉高 + 100ms 延时
```

## 3. 数据格式和数据处理

### 24 位数据帧格式

| 位域 | 位范围 | 长度 | 说明 |
|------|--------|------|------|
| D[13:0] | bit 23 ~ bit 10 | 14 bit | 角度数据（MSB 优先） |
| Mg[3:0] | bit 9 ~ bit 6 | 4 bit | 磁场状态 |
| CRC[5:0] | bit 5 ~ bit 0 | 6 bit | CRC 校验码 |

### 数据解析代码

```c
uint8_t buf[3] = {0};
HAL_SPI_Receive(&hspi1, buf, 3, 100);

/* 拼接 24 位原始数据 */
uint32_t raw = ((uint32_t)buf[0] << 16) |
               ((uint32_t)buf[1] << 8)  |
               ((uint32_t)buf[2]);

/* 提取各字段 */
uint16_t angle  = (raw >> 10) & 0x3FFF;    /* 14 位角度 */
uint8_t  status = (raw >> 6)  & 0x0F;      /* 4 位状态 */
uint8_t  crc    =  raw        & 0x3F;      /* 6 位 CRC */
```

### 角度换算

```c
/* 原始值范围：0 ~ 16383（14 位） */
/* 角度范围：0.0° ~ 360.0° */
float angle_deg = (float)angle * 360.0f / 16384.0f;
```

### 数据结构

```c
typedef struct {
    uint16_t raw_angle;     /* 原始角度值 0~16383 */
    float angle_deg;        /* 角度（度）0.0~360.0 */
    uint8_t status;         /* 磁场状态 0~15 */
    uint8_t crc;            /* CRC 校验码 0~63 */
    uint8_t crc_valid;      /* CRC 校验结果（当前未实现，固定为 1） */
} mt6701_data_t;
```

## 4. 注意事项

### 硬件相关

- **MISO/MOSI 短接**：本项目中 PA6 (MISO) 和 PA7 (MOSI) 硬件短接。必须将 SPI 配置为 `RXONLY` 模式，并将 PA7 重新配置为 `GPIO_MODE_INPUT`，否则 MOSI 推挽输出会干扰传感器数据，导致读取全 0。
- **波特率**：推荐 2~6 MHz。当前配置 170MHz / 64 = 2.656 MHz，在推荐范围内。
- **上电延时**：MT6701 上电后需等待至少 100ms 再读取数据。

### 时序相关

- **采样边沿**：MT6701 数据在 CLK 上升沿变化，必须在**下降沿**采样（SPI Mode 1, CPHA=1）。若使用 Mode 0（CPHA=0，上升沿采样），会采到数据变化的不稳定时刻，导致 status 位跳动。
- **CSN 延时**：CSN 拉低后需等待至少 2µs 再开始 SPI 传输。
- **DWT 延时**：使用 DWT 周期计数器实现微秒级精确延时，需在 `mt6701_init()` 中初始化 DWT。

### 软件相关

- **printf 浮点数**：STM32 默认链接的 `printf` 不支持 `%f`，需在 CMakeLists.txt 中添加链接标志：
  ```cmake
  target_link_options(${CMAKE_PROJECT_NAME} PRIVATE -u _printf_float)
  ```
  注意：会增加约 10KB Flash 占用。
- **CRC 校验**：当前驱动未实现 CRC 校验验证（`crc_valid` 固定为 1），如需数据可靠性保证，应补充 CRC-6 校验算法。
- **CubeMX 覆盖**：SPI 覆盖配置必须放在 `USER CODE BEGIN SPI1_Init 2` 块中（CubeMX 默认值之后），并重新调用 `HAL_SPI_Init()`，否则会被 CubeMX 默认值覆盖。
